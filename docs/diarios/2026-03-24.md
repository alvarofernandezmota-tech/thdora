## 🧠 Decisiones importantes del día

### Decisión 1 — thdora es el repo principal

`thdora` queda como **proyecto activo** y lugar donde se implementará la nueva evolución del sistema.

### Decisión 2 — thea-ia queda intacto

`thea-ia` **no se borra, no se archiva y no se toca**. Se mantiene como:
- proyecto histórico
- evidencia del trabajo hecho
- referencia de código para futuras fases
- posible línea de evolución propia en el futuro

### Decisión 3 — Opción A aceptada

La relación entre ambos proyectos queda definida así:

> `thea-ia` es la universidad. `thdora` es la carrera profesional.

Eso significa:
- thdora absorbe el aprendizaje de thea-ia
- no se copia código en bloque
- se consulta cada módulo de thea-ia cuando llegue su fase
- se reescribe con criterio y arquitectura limpia

### Decisión 4 — documentarlo todo y conectarlo todo

Regla del ecosistema a partir de hoy:

> **Si no está documentado, no existe.**

Y además:
- toda decisión importante → ADR
- todo cambio importante → CHANGELOG
- todo día trabajado → diario técnico
- toda documentación → enlazada entre sí
