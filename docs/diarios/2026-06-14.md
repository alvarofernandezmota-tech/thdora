## 🌙 Sesión noche — Fix Groq 401 + upgrade modelo

> Hora: 22:30 – 23:35 CEST

### Problema
Todas las llamadas al NLP (`groq_router.py`) fallaban con `401 Invalid API Key`:
```
ERROR — Error clasificando intent: Error code: 401 - {'error': {'message': 'Invalid API Key'}}
```

### Diagnóstico
1. `docker compose exec bot env | grep GROQ` → la variable existía en el contenedor
2. Verificación desde **Google Colab** con `requests.get` a `https://api.groq.com/openai/v1/models`
3. Key antigua → `401`. Key nueva generada → `200` con lista de modelos ✅
4. Modelo `llama3-8b-8192` detectado como **deprecado** (no aparece en catálogo Groq junio 2026)

### Fixes

**1. Rotación de key** — key anterior revocada en console.groq.com, nueva key en `.env`

**2. Modelo actualizado en `.env`:**
```env
# ANTES (deprecado, comentado):
# GROQ_MODEL=llama3-8b-8192

# DESPUÉS:
GROQ_MODEL=llama-3.3-70b-versatile
```

**3. Recarga forzada del entorno:**
```bash
docker compose down && docker compose up -d
```
> `docker restart` NO recarga `.env`. Solo `down && up` fuerza la recarga completa.

### Modelos Groq disponibles (free tier, junio 2026)

| Modelo | Contexto | Uso |
|--------|----------|-----|
| `llama-3.3-70b-versatile` ✅ | 128k | **Seleccionado** — NLP THDORA |
| `llama-3.1-8b-instant` | 128k | Rápido, ligero |
| `meta/llama-4-scout-17b` | 512k | Contexto muy largo |
| `qwen/qwen3-32b` | 32k | Razonamiento alternativo |

Límites free: 30 RPM / 6.000 TPM / 14.400 RPD — suficiente para uso personal.

### Git — migración a SSH
Durante la sesión se migró el remote de HTTPS a SSH:
```bash
git remote set-url origin git@github.com:alvarofernandezmota-tech/thdora.git
```
La clave `~/.ssh/id_ed25519_github` ya estaba configurada y funciona con passphrase.

### Aprendizajes

| Aprendizaje | Detalle |
|---|---|
| `docker restart` vs `down/up` | `restart` no recarga `.env`. Usar siempre `down && up` al cambiar variables |
| Verificar API keys aislada | Google Colab + `requests` permite aislar si el problema es la key o Docker |
| Modelos Groq deprecados | `llama3-8b-8192` eliminado. Usar `llama-3.1-8b-instant` o `llama-3.3-70b-versatile` |
| `git pull --rebase` | Evita merge commits innecesarios cuando el remote tiene commits nuevos |
