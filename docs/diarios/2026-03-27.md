## ✅ Logros — Sesión 2 (19:00–21:00)

### `src/bot/api_client.py` — CREADO

Cliente HTTP asíncrono con `httpx.AsyncClient`. Clase `ThdoraApiClient` con métodos:

| Método | Ruta API | Descripción |
|--------|----------|-------------|
| `health()` | `GET /health` | Check de disponibilidad |
| `get_appointments(date)` | `GET /appointments/{date}` | Lista citas del día |
| `create_appointment(date, time, type, notes)` | `POST /appointments/{date}` | Crea cita |
| `delete_appointment(date, index)` | `DELETE /appointments/{date}/{index}` | Borra por índice |
| `get_habits(date)` | `GET /habits/{date}` | Lista hábitos → convierte a `Dict` |
| `log_habit(date, habit, value)` | `POST /habits/{date}` | Registra hábito |
| `get_summary(date)` | `GET /summary/{date}` | Resumen completo |

**Por qué así:** La API usa rutas con fecha en el path (`/appointments/{date}`), no en body ni query params. El cliente normaliza la respuesta de hábitos (lista → dict) para que los handlers no tengan que preocuparse del formato.

**Clase `ApiError`:** Excepción custom para distinguir errores de red de errores de lógica.

### `src/bot/handlers.py` — CREADO

Handlers completos para python-telegram-bot >= 21. Todos async.

**Comandos simples:**
- `/start` — presentación + lista de comandos
- `/citas [YYYY-MM-DD]` — ver citas del día (hoy por defecto)
- `/habitos [YYYY-MM-DD]` — ver hábitos del día
- `/borrar <id>` — borrar cita por UUID
- `/resumen [YYYY-MM-DD]` — citas + hábitos juntos
- `/cancelar` — abortar conversación

**ConversationHandler `/nueva` (4 pasos):**
1. Fecha (`YYYY-MM-DD` o `hoy`)
2. Hora (`HH:MM` validado con regex)
3. Tipo (teclado inline: médica / personal / trabajo / otra)
4. Notas (texto libre o `/skip`)

**ConversationHandler `/habito` (2 pasos):**
1. Nombre (teclado inline con 7 hábitos comunes + "✏️ Otro…")
2. Valor (texto libre: `8h`, `30min`, `2L`, etc.)

**Rangos de estados separados** para evitar colisiones:
- `/nueva`: estados 0–3
- `/habito`: estados 10–11

## 🔍 Decisiones técnicas del día

### ¿Por qué `httpx` y no `aiohttp`?
`httpx` tiene una API casi idéntica a `requests` (familiar para Python junior), soporta async nativo, y viene incluido como dep transitiva de FastAPI. Menos fricción.

### ¿Por qué rangos de estados separados en ConversationHandler?
Si dos ConversationHandlers comparten el mismo número de estado (ej: ambos tienen estado `0`), python-telegram-bot puede confundirlos cuando los dos están activos. Rangos separados (0–3 y 10–11) eliminan esa colisión.

### ¿Por qué `load_dotenv()` en main y no en settings?
El bot no tiene módulo de configuración aún. `load_dotenv()` directo en `main.py` es la solución más simple y no introduce dependencias extra. Se puede refactorizar a `src/bot/config.py` en Fase 8.
